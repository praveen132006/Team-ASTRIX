Introduction

The ForensicAI API enables you to integrate our advanced AI-generated content detection capabilities into your applications, services, or workflows.

Our RESTful API provides endpoints for analyzing images, videos, and text to determine if they were generated by AI tools like DALL-E, Midjourney, Stable Diffusion, or ChatGPT. The API returns detailed forensic analysis results, manipulation probability scores, and specific markers that indicate AI generation.

Base URL
https://api.forensic-ai.com/v1

Authentication

All API requests must include your API key in the request headers. You can obtain an API key by registering for an account on our website. 

curl -X POST https://api.forensic-ai.com/v1/analyze/image \
-H "Authorization: Bearer YOUR_API_KEY" \
-H "Content-Type: multipart/form-data" \
-F "file=@/path/to/image.jpg"

Endpoints

Image Analysis

Analyze an image to detect if it was generated or manipulated by AI.

POST /analyze/image
Request

The request should be a multipart/form-data POST with the image file.

curl -X POST https://api.forensic-ai.com/v1/analyze/image \
-H "Authorization: Bearer YOUR_API_KEY" \
-H "Content-Type: multipart/form-data" \
-F "file=@/path/to/image.jpg"

Text Analysis

Analyze text to detect if it was generated by an AI language model.

POST /analyze/text
Request

The request should be a JSON POST with the text to analyze.

curl -X POST https://api.forensic-ai.com/v1/analyze/text \
-H "Authorization: Bearer YOUR_API_KEY" \
-H "Content-Type: application/json" \
-d '{
  "text": "The text content you want to analyze for AI generation..."
}'
Response

The response is a JSON object containing the analysis results.

{
  "ai_generated_probability": 0.78,
  "analysis_details": {
    "pattern_matches": {
      "moreover": 2,
      "in conclusion": 1,
      "with respect to": 3
    },
    "complexity_score": 0.65,
    "repetition_score": 0.42,
    "filler_word_score": 0.55
  },
  "analysis_method": "linguistic_analysis",
  "analysis_complete": true
}

Error Handling

The API uses conventional HTTP response codes to indicate the success or failure of requests.

Code Description
200 - OK Request succeeded. Response contains analysis results.
400 - Bad Request Invalid request, such as missing required parameters.
401 - Unauthorized Missing or invalid API key.
403 - Forbidden API key valid but doesn't have permission for the request.
429 - Too Many Requests Rate limit exceeded.
500 - Server Error Internal server error occurred during processing.
Error Response Format
{
  "error": {
    "code": "invalid_file_type",
    "message": "Unsupported file type. Please upload an image in JPEG, PNG, or WEBP format.",
    "status": 400
  }
}
